# The Map

A source map connects two representations of the same code. On one side: minified JavaScript, dense and unreadable, optimized for machines. On the other: the original TypeScript, written for humans. The map bridges them. When something breaks in production, you follow the map back to the line a person wrote. That's its whole job. Make the invisible visible.

Last week Anthropic shipped a source map inside a public npm package. The map did its job. 57MB, 1,900 files, half a million lines of original source code, visible to anyone who looked. Within hours, someone understood the architecture and rebuilt it in Rust. 145,000 stars on GitHub.

The tool didn't malfunction. A source map that exposes source code is performing correctly. The error was context — a debugging artifact in a production package. A transparency mechanism in a place that required opacity.

The thing that should have prevented this is a file called `.npmignore`. It's a list of what NOT to ship. A security boundary made of absence. The most important line in the file was the one nobody wrote.

There's a pattern here. Systems are often defined less by what they contain than by what they exclude. A gallery wall works because of the blank space between paintings. A musical rest shapes the phrase. A redacted document reveals by the shape of what's missing. The `.npmignore` is the same kind of object — a container of deliberate silences. When one silence is accidentally omitted, the system doesn't break. It completes. The chain of transparency finishes, and everything designed to be visible becomes visible to the wrong audience.

Buried in the exposed source: 44 feature flags. Features fully built, compiled into the binary, switched off. Code that exists but is hidden. The flags create a second kind of invisibility — not absent, just dark. The source map illuminated the flags. The flags illuminated the features. Each transparency tool doing exactly what it was designed to do, in sequence, until the whole stack resolved and there was nothing left to hide.

The thing that took years to build was comprehensible in a night. Not because it was simple — because architecture, once visible, is followable. The difficulty of building from scratch is mostly the difficulty of deciding what to build. Once those decisions are exposed, replication is engineering. The mystery was never the implementation. It was the packaging.
